Milliways

Milliways, noto anche come Il Ristorante alla Fine dell'Universo, è un esercizio immaginario descritto da Douglas Adams nel suo Guida Galattica per Autostoppisti. Nei sui vasti saloni (hall) il ristorante accoglie diverse razze, con requisiti diversi ed eventualmente incompatibili.

Milliways, also known as The Restaurant at the End of the Universe, is a fictional eating place described by Douglas Adams in his The Hitchhiker’s Guide to the Galaxy. In its vast halls, the restaurant accommodates different races, with different, possibly incompatible, requirements.

All classes must reside in the package named it.polito.oop.milliways.

Tutte le classi devono trovarsi nel packages it.polito.oop.milliways.

The class named Main in package main contains examples using the main methods.

La classe Main nel package main contiene esempi di uso del metodi principali.

È possibile accedere liberamente alla Documentazione JDK.

R1: Races

The facade class for the system is Restaurant through which all objects are created.

The class Race represents a race in the multiverse. Each race has a unique name, specified as argument to the factory method defineRace() in class Restaurant; if the same name is specified when creating two objects the exception MilliwaysException is thrown. The method returns an object of class Race. The method getName returns the name of the race.

A race has certain requirements. A requirements is a String, such as "Class M atmosphere (nitrogen-oxygen)", or "Low gravity (less than 2g)". The requirements for a given race are specified using the method addRequirement; if the same requirement is added more than once, the method throws the exception MilliwaysException. The method getRequirements returns the list of all requirements for the race, sorted alphabetically.

R1: Razze

La classe principale del sistema è Restaurant, tramite essa vengono creati tutti gli oggetti.

La classe Race rappresenta una razza nel multiverso. Ogni razza ha un nome unico, che viene passato come parametro al metodo di creazione defineRace() della classe Restaurant; se viene specificato un nome duplicato nella creazione viene lanciata un'eccezione MilliwaysException. Il metodo restituisce un oggetto di classe Race. Il metodo getName() restituisce il nome della razza.

Una razza ha certi requisiti. Un requisito è una String, tipo "Class M atmosphere (nitrogen-oxygen)", oppure "Low gravity (less than 2g)". I requisiti per una data razza sono specificati tramite il metodo addRequirement(); se lo stesso requisito è aggiunto più di una volta viene lanciata l'eccezione MilliwaysException. Il metodo getRequirements() restituisce la lista dei requisiti di un razza ordinati alfabeticamente.

R2: Party

The class Party represents a group of table companions. A new party can be created by means of the method createParty() in class Restaurant.

A party is composed of companions from different races. The method addCompanions() adds to the party a given number of companions of a certain race.

The method getNum() with no arguments returns the total number of companions in the group. When a race is specified it returns the number of companions in the group of that specific race.

The method getRequirements() return a list of all the requirements for the party, that is, the union of requirements of all the races composing the party, sorted in alphabetically and without repetitions.

The method getDescription() returns the composition of the party as a map having the races names as keys and their relative numbers as values.

R2: Gruppi

La classe Party rappresenta un gruppo di compagni di tavola. È possibile creare un nuovo gruppo tramite il metodo createParty() della classe Restaurant.

Un gruppo è composto da compagni di diverse razze. Il metodo addCompanions() aggiunge al gruppo un dato numero di compagni di una data razza.

Il metodo getNum() senza argomenti restituisce il numero di compagni nel gruppo. Quando viene passata una razza restituisce il numero di compagni di tale razza nel gruppo.

Il metodo getRequirements() restituisce una lista con tutti i requisiti del gruppo , ovvero, l'unione dei requisiti di tutte le razze che compongono il gruppo, ordinati alfabeticamente e senza ripetizioni.

Il metodo getDescription() restituisce la composizione del gruppo come mapp avente come chiavi i nomi della razze e come valori le corrispondenti numerosità.

R3: Hall

The class Hall represents a hall at Milliways. Halls have infinity capacity and a certain number of facilities. Each hall has a unique numeric id, specified when an object is created using method defineHall() in class Restaurant. If the same id is specified when creating two objects the exception MilliwaysException is thrown; the method getId() returns the id of a hall.

The method addFacility() add a facility to the hall. A facility is represented by a String, for instance "Class X atmosphere (methane)"; if the same facility is added more than once, the method throws the exception MilliwaysException. The method getFacilities() returns the list of all facilities available in the hall, sorted alphabetically.

The method isSuitable check whether a party may be accommodated in the hall, that is, all its requirements are satisfied by the hall’s facilities. A facility meets a requirement when the strings describing them are equal.

R3: Saloni

La classe Hall rappresenta un salone del Milliways. I saloni hanno capacità infinita ed un certo numero di servizi. Ogni salone ha un codice numerico univoco id, specificato quando viene creato tramite il metodo defineHall() della classe Restaurant. Se lo stesso id viene specificato più volte viene lanciata un'eccezione MilliwaysException; il metodo getId() della classe Hall restituisce il codice univoco del salone.

Il metodo addFacility() permette di aggiungere un servizio ad un salone. Un servizio è rappresentato da una String, per esempio "Class X atmosphere (methane)"; se lo stesso servizio viene aggiunto più volte allo stesso salone viene lanciata l'eccesione MilliwaysException. Il metodo getFacilities() restituisce la lista di tutti i servizi disponibili nel salore, ordinati alfabeticamente.

Il metodo isSuitable verifica se un gruppo può essere ospitato in un salone, ovvero se tutti i suoi requisiti sono soddisfatti dai servizi offerti dal salone. Un servizio soddisfa un requisito se le stringhe che li descrivono sono uguali.

R4: Restaurant

The class Restaurant represent the whole Milliways, and it is used to record the parties.

The method getHallList() returns the list of all halls, in the order they have been inserted.

The method seat() called with two arguments, accommodates the given party in the given hall; if the hall was not suitable for the party (i.e. isSuitable() is false), the exception MilliwaysException is thrown. When seat is invoked with only the party as argument, it scans all the halls in the restaurant and accommodates the party in the first suitable one. Halls are checked in order they have been added, and if not suitable hall is found, the exception MilliwaysException is thrown. Both methods returns the hall where the party has been successfully seated.

R5: Stats

Class Restaurant provides a few methods to compute stats on the Milliways.

The method statComposition() summarizes compositions of the parties, that is, the beings that were seated at the restaurant, according to their race. The method returns a Map with the race as key, and the total number of table companions recorded so far in seated parties.

The method statFacility() reports all facilities available in the restaurant, sorted by the descending number of halls where they are available and then alphabetically.

The method statHalls() returns a Map having as key a number of facilities and as value the list of halls codes providing that number of facilities. Keys and halls are in ascending order.

R4: Ristorante

La classe Restaurant rappresenta l'intero Milliways, ed è utilizzata per sistemare i gruppi.

Il metodo getHallList() restituisce la lista dei saloni nell'ordine con cui sono stati creati.

Il metodo seat() accetta due parametri e alloca il gruppo dato nel salone indicato; se il salone non può ospitare il gruppo (cioè isSuitable() è false), viene lanciata un'eccezione MilliwaysException. Quando seat() viene invocato con un solo parametro (un gruppo), scandisce i saloni del ristorante e alloca il gruppo nel primo salone che lo può accogliere. I saloni sono verificati nell'ordine in cui sono stati creati, e se non viene trovato alcun salone adatto viene lanciata un'eccezione MilliwaysException. Entrambi i metodi restituiscono il codice del salone in cui il gruppo è stato effettivamente allocato.

R5: Statistiche

La classe Restaurant permette di calcolare alcune statistiche su Milliways.

Il metodo statComposition() riassume la composizione dei gruppi, ovvero gli esseri che sono stati seduti al ristorante, secondo la loro razza. Il metodo restituisce una Map con la razza come chiave, e il numero totale di esseri di tale razza presenti nei diversi gruppi che sono stati seduti.

Il metodo statFacility() restituisce tutti i servizi disponibili nel ristorante, ordinati in base al numero decrescente di saloni in cui sono disponibili e, a parità di numero, alfabeticamente.

Il metodo statHalls() restituisce una Map avente per chiave il numero di servizi disponibili e come valore la lista dei codici dei saloni che offrono tale numero di servizi. Sia le chiavi che i saloni sono in ordine crescente.
